# Project context for codex-pair

This is a small Node.js + TypeScript + Express HTTP server that exposes a todo CRUD API. Stated requirements from the spec the agent is implementing:

- **Express 5 + TypeScript strict mode + Zod** for input validation
- 4 endpoints: `GET /todos`, `POST /todos`, `PATCH /todos/:id`, `DELETE /todos/:id`
- **Persist todos to a JSON file (`todos.json`) on disk**
- **Server MUST handle concurrent requests safely — NO race conditions on the file**
- **`tsc --noEmit` must pass**
- **`vitest run` must pass**

When reviewing files in this project, the deployment shape to assume:
- The server can receive multiple HTTP requests in parallel
- The file system is shared (single mounted volume, no cluster)
- The Node process is single-threaded but I/O is async, so concurrent requests interleave
- Tests use supertest in-process — the concurrency-safety requirement is for production, not tests

Anything that would fail under concurrent load, type-check, or fail a fresh test run is HIGH severity per the codex-pair confidence ladder.
